# AudioQAS 网页端验收清单

更新时间：2026-05-19

## 1. 纯人声评测

### 1.1 单文件测评

- 进入 `纯人声评测`
- 点击 `单文件测评`
- 上传一段纯人声音频
- 确认页面展示：
  - 文件摘要为真实数据
  - `首屏结论` 与 `处理建议` 分开显示
  - `预处理追溯` 为详细链路
  - `DNSMOS` 时显示 `OVRL / SIG / BAK`
  - `NISQA` 时显示 `OVRL / NOI / DIS / COL / LOUD`
  - 信号分析数值已格式化，不出现过长浮点
  - `详细数据` 可切换：
    - `模型维度`
    - `信号分析`
    - `完整表格`

### 1.2 对比评测

- 进入 `纯人声评测`
- 点击 `对比评测`
- 上传至少 2 组文件
- 确认页面展示：
  - 有 `推荐版本`
  - 有 `推荐排序`
  - 自由对比与基准对比都可切换
  - 表格显示真实模型分数与信号指标
  - `NISQA` 对比时维度完整
  - 每组 `预处理追溯` 为详细链路

## 2. 综合音频分析

### 2.1 单文件分析

- 进入 `综合音频分析`
- 点击 `单文件分析`
- 上传一段混合音频或视频
- 确认页面展示：
  - 文件摘要为真实数据
  - `首屏结论` 与 `处理建议` 分开显示
  - `预处理追溯` 为详细链路
  - `AudioBox Aesthetics` 显示 `PQ / CE / CU / PC`
  - 信号分析数值已格式化
  - `详细数据` 可切换：
    - `模型维度`
    - `信号分析`
    - `完整表格`

### 2.2 对比分析

- 进入 `综合音频分析`
- 点击 `对比分析`
- 上传至少 2 组文件
- 确认页面展示：
  - 有 `推荐版本`
  - 有 `推荐排序`
  - 自由对比与基准对比都可切换
  - 表格显示真实模型分数与信号指标
  - 每组 `预处理追溯` 为详细链路

## 3. 历史

- 进入 `历史`
- 确认页面不是纯静态占位，而是从 `/api/history` 拉取列表
- 完成一次真实上传后切到 `历史`，无需手动刷新即可看到新记录
- 若有数据：
  - 显示时间
  - 显示页面类型
  - 显示模型
  - 显示文件摘要
  - 显示核心指标摘要
  - 显示追溯摘要
- 刷新页面后历史记录仍保留
- 若无数据：
  - 显示空状态

## 4. 设置

- 进入 `设置`
- 确认 `默认模型`、`预处理追溯`、`默认对比模式` 来自真实状态，不只是纯写死文本
- 切换 `纯人声评测` 当前模型后，返回设置页确认默认模型显示同步变化
- 切换 `预处理追溯` 开关，确认结果页/历史页追溯显示联动
- 切换 `默认对比模式`，确认新进入对比页时默认模式联动
- 刷新页面后，`预处理追溯` 与 `默认对比模式` 的修改仍保留

## 5. 通用验证

- 强制刷新页面后脚本能更新，不受旧缓存影响
- 上传失败时出现明确错误提示
- 进度条为阶段性伪进度，不是固定 35%
- 页面结构仍保持 `web-preview` 既有层级
