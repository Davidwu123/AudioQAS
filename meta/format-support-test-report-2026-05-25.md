# AudioQAS Format Support Test Report

Date: 2026-05-25

## Scope

This report covers product-promised upload formats generated by local `ffmpeg` and verified through the real AudioQAS upload + preprocessing path.

Verified audio formats:

- `wav`
- `flac`
- `mp3`
- `aac`
- `m4a`
- `ogg`

Verified video formats:

- `mp4`
- `mov`
- `mkv`
- `avi`

Not product-promised in this report:

- `wmv`
- `flv`
- arbitrary headerless raw PCM

## Fixture Location And Generation

Reusable fixtures are committed under:

```text
tests/fixtures/format_matrix/
```

Each fixture uses the filename pattern `format_matrix.<ext>`.

Audio fixtures were generated with:

```bash
ffmpeg -f lavfi -i sine=frequency=1000:duration=0.35:sample_rate=44100 <codec args> output.<ext>
```

Video fixtures were generated with:

```bash
ffmpeg -f lavfi -i testsrc=size=64x64:rate=10:duration=0.35 \
       -f lavfi -i sine=frequency=1000:duration=0.35:sample_rate=44100 \
       -shortest <codec args> output.<ext>
```

## Assertions

For every tested format, the integration test asserts:

- upload API returns HTTP `200`
- decoded/preprocessed duration is greater than zero
- channel count is present
- a preprocessing output file exists
- `mp3/aac/m4a` include `decode_audio` in `pipeline_steps`
- `mp4/mov/mkv/avi` include `source_video -> extract_audio` in `pipeline_steps`

## Remaining Boundaries

- `wmv/flv` are still backend-recognized video extensions, but not product-promised because they are not covered by this product matrix.
- Headerless raw PCM remains unsupported unless explicit PCM parameters are introduced.
- Damaged WAV headers are only recovered for controlled `.wav` PCM cases where `fmt ` metadata and payload bytes are still available.
